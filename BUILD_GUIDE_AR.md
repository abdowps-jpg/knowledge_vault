# دليل بسيط (بالعربي) لتشغيل وبناء المشروع من الصفر

> هذا الدليل مكتوب لغير المبرمجين: امشِ خطوة خطوة واعتبره "قائمة تنفيذ".

## 1) قبل أي شيء: الأدوات المطلوبة

لازم تكون مثبت على جهازك:

1. **Node.js 20 أو أحدث**
2. **pnpm** (مدير الحزم)
3. **Git**

### التأكد إن الأدوات موجودة

انسخ الأوامر التالية في الطرفية (Terminal):

```bash
node -v
pnpm -v
git --version
```

لو ظهر رقم إصدار لكل أداة، تمام.

---

## 2) تحميل المشروع وفتح المجلد

```bash
git clone <REPO_URL>
cd knowledge_vault
```

> بدل `<REPO_URL>` برابط المشروع الحقيقي.

---

## 3) تثبيت المكتبات (Dependencies)

```bash
pnpm install
```

دي أهم خطوة أول مرة. ممكن تاخد شوية وقت.

---

## 4) ضبط متغيرات البيئة (.env)

المشروع غالبًا يحتاج ملف إعدادات اسمه `.env` (مثل: قاعدة البيانات، OAuth، مفاتيح API).

### أسرع طريقة

1. ابحث داخل المشروع عن ملف اسمه مثل:
   - `.env.example`
   - `.env.sample`
2. انسخه لملف جديد اسمه `.env`
3. عبّي القيم المطلوبة.

مثال (تخيلي):

```bash
cp .env.example .env
```

> لو مفيش ملف مثال، اطلب من صاحب المشروع قيم `.env` الجاهزة.

---

## 5) تشغيل قاعدة البيانات (لو مطلوبة)

لو المشروع مربوط بقاعدة بيانات، نفّذ:

```bash
pnpm db:push
```

ده بيجهز الجداول (migrations/schema) حسب الكود الحالي.

---

## 6) تشغيل المشروع أثناء التطوير

```bash
pnpm dev
```

المشروع هنا غالبًا يشغل:
- **Back-end server**
- **Expo/Web front-end**

لو اشتغل بنجاح، افتح الرابط اللي يظهر في الطرفية (عادة Web على بورت مثل `8081`).

---

## 7) فحوصات قبل البناء النهائي

نفّذ الفحوصات دي بالترتيب:

```bash
pnpm check
pnpm lint
pnpm test
```

لازم تكون النتائج بدون أخطاء (أو أقل أخطاء ممكنة قبل الإطلاق).

---

## 8) بناء نسخة الإنتاج (Production Build)

### بناء السيرفر

```bash
pnpm build
```

### تشغيل نسخة الإنتاج

```bash
pnpm start
```

لو ظهرت رسالة إن السيرفر بدأ بدون أخطاء، فالبناء ناجح.

---

## 9) معنى أهم الأوامر بسرعة

- `pnpm dev`: تشغيل وضع التطوير.
- `pnpm check`: فحص TypeScript.
- `pnpm lint`: فحص جودة الكود.
- `pnpm test`: تشغيل الاختبارات.
- `pnpm build`: تجهيز نسخة إنتاج.
- `pnpm start`: تشغيل نسخة الإنتاج.

---

## 10) أشهر المشاكل وحلولها

### مشكلة: `pnpm: command not found`
ثبت pnpm:

```bash
npm install -g pnpm
```

### مشكلة: منفذ مستخدم بالفعل (Port in use)
اقفل البرنامج اللي شغال على نفس البورت أو غيّر البورت من إعدادات التشغيل.

### مشكلة: نقص في `.env`
أي خطأ فيه كلمات مثل `DATABASE_URL` أو `OAUTH` غالبًا يعني قيمة ناقصة في ملف `.env`.

### مشكلة: فشل في `pnpm install`
جرّب:

```bash
pnpm store prune
rm -rf node_modules pnpm-lock.yaml
pnpm install
```

---

## 11) خطة تنفيذ جاهزة (Copy/Paste)

لو عايز تمشي بسرعة وبالترتيب:

```bash
pnpm install
pnpm db:push
pnpm check
pnpm lint
pnpm test
pnpm build
pnpm start
```

---

## 12) لو أنت لسه مبتدئ جدًا

أقدر أساعدك كمان بخطة 1:1 حسب جهازك:
- Windows أو Mac أو Linux
- هل هتشغل Web فقط ولا Android/iOS
- هل معاك ملف `.env` ولا لأ

وابعت لي أول خطأ يظهر لك في الطرفية وأنا أقولك تصلحه خطوة بخطوة.
